📋 PROBLEMA ORIGINAL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

El sistema FFA detecta cuando el peso es inestable y NO permite realizar
la captura. El emulador anterior generaba peso con variación constante,
por lo que nunca se consideraba estable.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ SOLUCIÓN IMPLEMENTADA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

El emulador ahora:

1️⃣  MANTIENE PESO ESTABLE POR 1.2 SEGUNDOS
   - El peso se genera y se mantiene estable durante 1.2s
   - Solo agrega ruido mínimo (±0.05g) durante estabilidad
   - Después de 1.2s, genera un nuevo peso estable
   
2️⃣  SIMULA COMPORTAMIENTO REALISTA
   - Variación de peso cada 1.2s (simula diferentes objetos)
   - Ruido pequeño durante estabilidad (simula sensor real)
   - Peso estable permite que el sistema detecte peso válido
   
3️⃣  MANEJO INTELIGENTE DE TARA Y ZERO
   - Después de setTare(): peso va a ~0g y se mantiene estable
   - Después de setZero(): genera nuevo peso estable
   - Después de force_new_stable_weight(): nuevo peso inmediato

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
